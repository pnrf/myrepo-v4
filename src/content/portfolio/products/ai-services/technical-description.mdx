# Техническое описание

## Место и способ применения

1. На автоматизированном рабочем месте (АРМ) врача-рентгенолога.
2. В медицинской информационной системы ФГБУ «Лечебный центр».
3. В облачной инфраструктуре ООО «РадиМедИИ».

## Принцип действия

Изображения, полученные на компьютерном томографе, поступают и хранятся на PACS-сервере медицинской организации. Врач-рентгенолог через графический пользовательский интерфейс выбирает требуемые изображения. ИИ-сервис через интеграционный шлюз обращается к PACS-серверу, копирует выбранные изображения, анонимизирует их и передает на обработку в нейронную сеть, получает и выводит результаты на АРМ врача-рентгенолога.

## Ожидаемый диагностический эффект 

Повышение качества работы врачей-рентгенологов, минимизация терминологических расхождений при заполнении протокола исследования, ошибок интерпретации, в том числе за счет использования ИИ-сервиса в системе "двойного чтения".

## Ожидаемый экономический эффект

Уменьшение времени на подготовку протокола диагностического исследования, опосредованное снижение времени проведения исследования. Экономический расчет прилагается (Приложение №4).

## Этапы разработки

1. Составление технического задания (ТЗ) на разработку ИИ-сервиса.
2. Разметка анонимизированных DICOM-изображений.
3. Составление из них структурированного набора данных (датасета), необходимого для обучения нейронной сети.
4. Обучение нейронной сети, изготовление и тестирование ИИ-сервиса.
5. Приемка результатов.

## Требования к обработке исследований

1. Исходная серия с диагностическими изображениями остается неизменной в результате обработки исследования ИИ-сервисом.
2. При выборе изображения/серии для анализа ИИ-сервисом целевой орган (предстательная железа) представлен на изображении или в серии в полном объеме; выбранные окна (W/L): T2 сагинальная + T2 аксиальная; толщина среза: `<= 4 мм`. Если целевой орган не охвачен в полном объеме, ИИ-сервис информирует об этом сообщением об ошибке ("Series error").
3. Погрешность измерения для линейных и объемных величин не превышает 5% от экспертной разметки. Отклонение при изменении угловых величин не превышает 2 (двух) градусов.

## Требования к конечному результату

Взаимодействие с ИИ-сервисом осуществляется через графический пользовательский интерфейс, отображаемый на АРМ врача-рентгенолога. В результате обработки исследования ИИ-сервис формирует:
-	Текстовое заключение в формате DICOM SR, на основании которого возможно формирование электронной медицинской записи (предзаполненный протокол диагностического исследования);
-	Дополнительную серии изображений, содержащую результаты обработки исследований ИИ-сервисом (с контурированием находок и/или наложением масок). 
-	Общая вероятность патологии в исследовании в целом передается в системном сообщении в диапазоне от 0 до 100. 
-	Вероятность патологии в исследовании в целом и отдельно для каждой из целевых патологий является обязательным полем и заполнено для всех обработанных исследований.
-	Перечень полей, доступных для заполнения и (или) редактирования в DICOM SR, указана в Таблице 1.

| Название поля                                          | Формат значения                                        |
|-------------------------------------------------------|------------------------------------------------------|
| Уникальный номер исследования                        | текст                                               |
| Уникальный номер дополнительной серии изображений    | текст                                               |
| Наличие патологии в исследовании                    | true/false                                          |
| Вероятность наличия патологии во всем исследовании в целом | целое число в диапазоне 0–100                      |
| Номер модели ИИ-сервиса                             | текст                                               |
| Версия ПО ИИ-сервиса                                | текст                                               |
| Описание                                            | текст                                               |
| Заключение                                          | текст                                               |
| Время начала загрузки исследования из PACS          | ISO 8601. Шаблон: YYYY-MM-DDThh:mm:ss.sss+hhmm    |
| Время окончания загрузки исследования из PACS       | ISO 8601. Шаблон: YYYY-MM-DDThh:mm:ss.sss+hhmm    |
| Время начала обработки исследования на стороне ИИ-сервиса | ISO 8601. Шаблон: YYYY-MM-DDThh:mm:ss.sss+hhmm    |
| Время окончания обработки исследования на стороне ИИ-сервиса | ISO 8601. Шаблон: YYYY-MM-DDThh:mm:ss.sss+hhmm    |

## Требования к DICOM SR

Разделы "Описание" и "Заключение" должны содержать сведения:
-	о размере предстательной железы (в мм): сагиттальный (передне-задний), фронтальный (поперечный), вертикальный (продольный). Значение: целое число;
-	об объеме предстательной железы (в см3). Значение: целое число.
-	о вероятности наличия патологии. Значение: дробное число в диапазоне от 0,00 до 1,00.

## Требования к отображению находок

-	отдельные находки, а также области/зоны маркируются выделением контура;
-	применяются маски (контур с цветовой заливкой);
-	отображается числовая индикация находок;
-	отображается значение вероятности наличие патологии в диапазоне от 0,00 до 1,00.
-	изображения в серии, содержащие патологические находки, обозначены на отдельной полосе визуализации уровня просмотра серии в дополнительной серии (ScrollBar);
-	для обозначения рутинных изменений предстательной железы используется аббревиатура "PROSTATE-MEAS".
