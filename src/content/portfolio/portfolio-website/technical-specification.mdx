# Техническое задание

## Введение

Настоящее техническое задание определяет основные требования к персональному сайту-портфолио технического писателя.

**Полное наименование сайта:** Мой репозиторий. Портфолио технического писателя  
**Краткое наименование сайта:** Мой репозиторий  
**Краткая характеристика области применения:** Мой репозиторий – персональный сайт технического писателя. На сайте публикуются сведения о профессиональном опыте его автора, портфолио, база знаний и блог.

## Основания для разработки

Разработка сайта выполняется, исходя из внутренних убеждений и профессионального опыта его автора. По этой причине документ-основание для разработки отсутствует.

## Назначение разработки

**Эксплуатационное назначение:** 

Сайт предназначен для личных некоммерческих целей его автора: формирования личного бренда, поиска работы, подготовки к собеседованию, а также для систематизации знаний о технологиях, применяемых в профессиональной деятельности технического писателя и смежных областях.

**Функциональное назначение:**

Администратор сайта может добавлять, редактировать и удалять контент, управлять структурой и набором функций сайта, настраивать параметры его работы, изменять программный код.

Пользователям сайта доступны просмотр контента и поиск информации по сайту. Личный кабинет, формы отправки данных и комментирование записей не предусмотрены.

## Требования к программе

**Требования к функциональным характеристикам:**

Сайт должен быть разработан на фреймворке Nextra (https://nextra.site), который применяет технологию SSG (Static Site Generation).

Функциональные харакретистики сайта определяются возможностями указанного фреймворка. В обязательном порядке должна быть реализована функция "Поиск по сайту". 

**Требования к надежности:**

Сайт должен быть доступен ежедневно в круглосуточном режиме за исключением времени, необходимого на проведение профилактических работ и устранение аварийных (сбойных) ситуаций.

Сайт не должен иметь технических ограничений, связанных с личностью пользователя, его гражданской принадлежностью, вероисповеданием и местонахождением.

Требования по безопасности должны быть реализованы на уровне регистратора доменного имени, хостинг-провайдера, а также публичного репозитория, в котором хранятся файлы сайта.
