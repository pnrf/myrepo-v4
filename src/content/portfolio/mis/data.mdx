# Данные

Подсистема передачи и хранения изображений должна соответствовать следующим требованиям:

1. **Общие требования к подсистеме передачи, хранения изображений и интеграционного взаимодействия с системами-поставщиками DICOM-исследований:**
   - Обеспечение поддержки передачи DICOM-изображений без потерь качества.
   - Возможность настройки перемещения исследований в различные PACS в соответствии с значениями тегов DICOM.
   - Поддержка протокола DICOM версии 3.0.
   - Реализация поддержки основных классов DICOM SOP.
   - Передача DICOM-объектов по расширению стандарта DICOM – WADO.
   - Возможность многопоточного соединения с DICOM-сервером.
   - Поддержка работы с несколькими DICOM-серверами.

2. **Поддержка следующих объектов стандарта DICOM 3.0 в качестве провайдера (SCP):**
   - Computed Radiography Image Storage.
   - Digital X-Ray Image Storage – For Presentation.
   - Digital X-Ray Image Storage – For Processing.
   - Digital Mammography X-Ray Image Storage – For Presentation.
   - Digital Mammography X-Ray Image Storage – For Preocessing.
   - Digital Intra-oral X-Ray Image Storage – For Presentation.
   - Digital Intra-oral X-Ray Image Storage – For Processing.
   - CT Image Storage.
   - Secondary Capture Image Storage.
   - X-Ray Angiographic Image Storage.
   - X-Ray Radiofluoroscopic Image Storage.
   - RT Image Storage.
   - Basic Text SR.
   - Enhanced SR.
   - Comprehensive SR.
   - Key Object Selection Document.
   - Storage Commitment Push Model.
   - Patient Root Q/R – FIND.
   - Patient Root Q/R – MOVE.
   - Study Root Q/R – FIND.
   - Study Root Q/R – MOVE.
   - Patient/Study Only Q/R – FIND.
   - Patient/Study Only Q/R – MOVE.
   - Verification.

3. **Дополнительные требования:**
   - Поддержка архивирования не-DICOM объектов в DICOM-контейнерах.
   - Реализация функциональных возможностей рабочих мест пользователей системы как web-приложения, посредством интернет-браузера.
   - Использование СУБД со

**2. Функциональные требования**

Подсистема обязана обеспечивать выполнение следующих функций:

* **Передача данных.** Реализация механизма передачи DICOM-изображений с гарантированным сохранением их исходного качества без каких-либо потерь при транспортировке.

* **Интеграционное взаимодействие.** Настройка автоматического перемещения медицинских исследований в различные системы PACS на основании значений тегов DICOM.

* **Протокольная поддержка.** Полная реализация протокола DICOM версии 3.0, включая основные классы DICOM SOP.

* **Расширенная функциональность.** Поддержка передачи DICOM-объектов посредством расширения стандарта WADO, а также обеспечение многопоточного соединения с DICOM-сервером.

**3. Технические требования**

Подсистема должна соответствовать следующим техническим параметрам:

* **Мультисерверная работа.** Возможность одновременного взаимодействия с несколькими DICOM-серверами.

* **Поддержка объектов.** Реализация всех необходимых объектов стандарта DICOM 3.0 в качестве провайдера (SCP), включая:
  * Хранение изображений компьютерной рентгенографии
  * Хранение цифровых рентгеновских изображений
  * Хранение маммографических изображений
  * Хранение компьютерной томографии
  * Хранение ангиографических изображений
  * Хранение радиофлюороскопических изображений
  * Хранение радиологических изображений
  * Поддержка различных текстовых форматов
  * Реализация моделей хранения и верификации

* **Архивация данных.** Возможность архивирования не-DICOM объектов в DICOM-контейнерах.

**4. Требования к инфраструктуре**

Подсистема должна обеспечивать:

* **Веб-доступность.** Реализация пользовательских рабочих мест в формате веб-приложений, доступных через стандартный интернет-браузер.

* **База данных.** Использование системы управления базами данных (СУБД) с открытой лицензией.

* **Виртуализация.** Возможность установки на виртуальные машины.

* **Масштабируемость.** Отсутствие ограничений на:
  * Количество одновременно подключенных пользователей к каждому DICOM-серверу
  * Объём обрабатываемых исследований
  * Количество исследований за определённый период времени

**5. Требования к подсистеме машинной диагностики патологий лёгких**

Подсистема должна обеспечивать:

* **Предварительная диагностика.** Проведение автоматизированного анализа данных, полученных с рентгенографических диагностических аппаратов.

* **Обучение системы.** Возможность обучения на размеченных DICOM-сериях изображений.

* **Обработка данных.** Поддержка DICOM-серий с параметрами:
  * Размер кадра не менее 1024×1024 пикселей
  * Цветовое разрешение от 12 до 16 бит

* **Выявление патологий.** Автоматическое обнаружение следующих признаков:
  * Плевральный выпот
  * Пневмоторакс
  * Ателектаз
  * Очаги затемнения
  * Инфильтрация/консолидация
  * Диссеминация
  * Полости с распадом
  * Полости с уровнем жидкости
  * Кальцинаты
  * Нарушения целостности кортикального слоя

**6. Процесс обработки исследований**

Подсистема должна реализовывать следующие этапы обработки:

* **Подготовка данных.** Проверка корректности входных параметров исследования.

* **Аналитический этап.** Проведение комплексного анализа DICOM-исследования.

* **Формирование результатов.** Создание аннотированной серии и файла отчёта, включающих:
  * Рекомендацию по состоянию здоровья
  * Визуальное выделение подозрительных областей
  * Текстовую информацию в формате DICOM SR

**7. Требования к отчётности**

Результаты анализа должны представляться в следующем формате:

* **Аннотированная серия.** Копия исходного исследования с добавленными результатами анализа.

* **Файл отчёта.** Текстовый документ в формате CSV, содержащий:
  * Идентификатор серии исследования
  * Рекомендацию системы
  * Временные метки
  * Информацию об ошибках при их наличии
