# ИТ архитектура

## Базовая структура

* Базовая архитектура построена на собственной ИТ-инфраструктуре
* Имеется возможность обработки данных в удалённом ЦОД

## Основные компоненты архитектуры

* Подсистема предоставления данных
* Подсистема передачи данных
* Подсистема обработки данных
* Подсистема хранения данных
* Инженерная инфраструктура ИТ
* Система защиты информации
* Подсистема диспетчеризации инженерных систем
* Подсистема медицинского оборудования с сетевыми интерфейсами

## Контурная организация

ИТ-инфраструктура разделена на два основных контура:

* Контур медицинского оборудования с системами обработки данных
* Контур информационных систем (медицинские, административные, базовые ИТ-системы)

## Взаимодействие компонентов

* Межконтурное взаимодействие осуществляется через межсетевые экраны
* Информационный обмен происходит по протоколу TCP/IP через локальную сеть
* Доступ пользователей реализуется через HTTPS
* Администрирование осуществляется через HTTPS, RDP, SSH

## Технические особенности

* Виртуализация применяется во всех компонентах, кроме систем с особыми технологическими требованиями
* Защита данных обеспечивается криптографическими средствами
* Мониторинг осуществляется централизованно
* Резервирование реализовано на всех ключевых компонентах

## Внешние интеграции

* ЕГИСЗ (Единая государственная информационная система в сфере здравоохранения)
* ВИМИС (Вертикально интегрированная медицинская информационная система)
* Системы страхования (ОМС, ДМС)
* ЛИС (лабораторная информационная система)
* РИС (радиологическая информационная система)
* Облачные ИИ-сервисы
* Системы анестезиологии и реанимации

## Режимы работы

* Основной режим — круглосуточная работа
* Профилактический режим — для обслуживания и модернизации
