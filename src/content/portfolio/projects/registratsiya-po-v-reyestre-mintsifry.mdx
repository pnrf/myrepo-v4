# Регистрация ПО в Минцифры

## Описание ситуации

Компания, в которой я работал, разрабатывала сервисы искусственного интеллекта. Для участия в госзакупках и получения преференций от государства решили зарегистрировать свои разработки в реестре Минцифры. Для этого привлекли внешних консультантов, имеющих релевантный опыт.

## Задача

Мне поручили помогать консультантам в заполнении заявлений, а также составлять необходимые документы.

## Исполнение

Заявления заполняли на сайте Госуслуг, подписывали их усиленной ЭЦП. К заявлениям прикрепляли файлы с документами. Вот некоторые из тех, над которыми я работал:

- [Руководство по эксплуатации ПО](../docs/rukovodstvo-po-ekspluatatsii-po.mdx)
- [Описание процессов жизненного цикла](../docs/opisaniye-protsessov-zhiznennogo-tsikla.mdx)
- [Описание функциональных характеристик ПО](../docs/opisaniye-funktsionalnykh-kharakteristik.mdx)
- [Образец лицензионного договора](../docs/obrazets-litsenzionnogo-dogovora.mdx)
- [Технические условия](../docs/tekhnicheskiye-usloviya.mdx)

При выполнении задачи руководствовался действующими нормативами и официальной документацией:

- [Единый реестр российских программ для ЭВМ и БД. Официальный сайт реестра](https://reestr.digital.gov.ru)
- [Классификатор программ для ЭВМ и БД. СПС Гарант](https://base.garant.ru/74832091/)
- [Методические рекомендации по подаче заявлений и уведомлений. Документ pdf](https://reestr.digital.gov.ru/upload/documents/Методичка_Подача_заявления.pdf)
- [Правила формирования и ведения единого реестра программ для ЭВМ и БД. СПС Гарант](https://base.garant.ru/71252170/)
- [Требования к правообладателям и программам. Официальный сайт Госуслуг](https://www.gosuslugi.ru/reestrpo)

## Результат

На момент моего увольнения из компании один из сервисов внесен в реестр ([ссылка на реестровую запись](https://reestr.digital.gov.ru/reestr/1910660/)). По другим не знаю.
